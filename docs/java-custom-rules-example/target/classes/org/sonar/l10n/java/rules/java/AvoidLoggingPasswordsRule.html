Passwords must not be logged.

Logging sensitive information such as passwords, even in encoded form (e.g. Base64), can expose vulnerabilities in logs and lead to security issues.

This rule raises an issue whenever a password is logged via log.*, even if it's encoded using Base64 or similar methods.

Noncompliant Code Example

class MyClass {
    int     foo3(int value) {
    String password = "PASSWORD";
    log.info("This log contains a password: {}", password); // Noncompliant
    return 0;
  }
}

Compliant Solution

class MyClass {
  int     foo1() {
    log.error("This is a valid test");
  }

  void    foo2(int value) {
    log.warn("This is another valid test");
  }

  void    foo2(int value) {
    log.info("This is another valid test");
  }
}
